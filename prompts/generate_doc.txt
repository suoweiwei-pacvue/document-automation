# {module_title} 功能逻辑文档

## 1. 模块概述
- 详细描述模块的职责与定位
- 说明在系统架构中的位置和上下游关系
- 列出涉及的后端模块（Maven 模块名）和前端组件
- 说明 Maven 坐标和部署方式（如果可确认）
- 主要参考：**代码片段**中的包名、类名、模块结构

## 2. 用户视角
- 基于 **PRD 文档**详细描述所有功能场景，逐个场景展开
- 描述用户操作流程（步骤 1、步骤 2、...）
- 说明 UI 交互要点（按钮、弹窗、表单、列表等）
- 如有 **Figma 设计稿**信息，结合设计稿描述页面布局、组件名称、交互元素
- 交叉验证：PRD 中描述的功能在代码中是否有对应实现

## 3. 核心 API
- 列出所有 REST 端点，格式为：
  - **路径**: `GET/POST /api/xxx`
  - **参数**: 请求参数及类型
  - **返回值**: 响应结构
  - **说明**: 业务含义
- 说明前端调用方式和对应的 API 文件
- 主要参考：**代码片段**中的 @RequestMapping/@GetMapping/@PostMapping

## 4. 核心业务流程
- 用文字**逐步详细描述**主要业务流程，不要概括，要展开每一步的具体逻辑
- 包含 Mermaid 流程图（flowchart TD 或 LR）
- 说明数据流转路径：前端请求 → Controller → Service → 下游调用 → 返回
- 解释关键设计模式（策略模式、责任链等）
- 主要参考：**代码片段** + **技术评审文档**

## 5. 数据模型
- 描述数据库表结构（表名、关键字段、索引）
- 列出核心 DTO/VO/枚举，说明每个字段的含义
- 说明 ChartSetting JSON 结构（如果适用）
- 主要参考：**代码片段**中的实体类和 MyBatis Mapper

## 6. 平台差异（如果适用）
- 说明各平台（Amazon/Walmart/DSP/...）的特殊处理逻辑
- 描述指标映射关系
- 列出平台特有的配置和枚举值
- 主要参考：**代码片段** + **技术评审文档**

## 7. 配置与依赖
- 列出关键配置项（Apollo/application.yml）
- 说明 Feign 下游服务依赖及其接口
- 描述缓存策略（Redis key 格式、过期时间）
- 描述消息队列使用（Kafka topic、消费逻辑）
- 主要参考：**代码片段**中的 @FeignClient、@Value、@Cacheable

## 8. 版本演进
- 基于**技术评审文档**整理主要版本变更，按时间线排列
- 列出待优化项和技术债务
- 主要参考：**技术评审文档**

## 9. 已知问题与边界情况
- 列出代码中的 TODO、FIXME 注释
- 说明异常处理和降级策略
- 描述并发、超时等边界情况

## 写作要求
- 面向研发、测试、产品三类读者
- **事无巨细**：每个 API 的参数、每个流程的分支、每个配置项都要写清楚
- **代码中的事实必须准确**，类名、方法名、路径必须与代码片段一致
- 不确定的信息标注"**待确认**"
- 使用中文撰写
- 每个主要流程都要包含 Mermaid 流程图
- 不要空泛的最佳实践，只写项目实际的逻辑
- 文档篇幅目标：**8000-15000 字**，宁多勿少
