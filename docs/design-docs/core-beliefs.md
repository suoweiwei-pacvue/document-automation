# 核心设计信念

## 1. 准确性 > 完整性 > 结构

文档准确性是第一优先级。宁可标注"待确认"，不可臆造代码中不存在的事实。

## 2. 代码是唯一真相来源

生成的文档必须可追溯到源代码。PRD 和技术评审仅作为补充上下文，不作为逻辑真相。

## 3. 多源交叉验证

后端代码提供逻辑准确性，前端代码提供用户视角完整性，Confluence 文档提供历史上下文。三者交叉比对提升可信度。

## 4. 按模块独立生成

每个文档模块独立生成、独立验证。模块间不假设依赖关系。

## 5. 可迭代的 Prompt 设计

Prompt 模板存放在独立目录，与代码解耦，支持快速迭代优化而不改动核心代码。
