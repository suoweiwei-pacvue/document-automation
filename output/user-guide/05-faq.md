# 常见问题

## 功能简介

本章节汇总了 Pacvue Custom Dashboard 在日常使用过程中最常遇到的问题与解答，涵盖 Dashboard 创建与管理、图表配置、数据源选择、分享协作、下载导出等各个环节。无论您是初次使用还是已有一定经验，都可以在这里快速找到解决方案。

适用场景：当您在操作过程中遇到疑问、报错或不确定某个功能的用法时，可先在本章节中查找对应问题。

---

## 使用场景

- 创建 Dashboard 或图表时遇到选项灰置、无法保存等问题，需要排查原因
- 不确定某种图表类型应该选择哪种模式或配置
- 切换平台后发现筛选器或指标列表发生变化，想了解原因
- 分享 Dashboard 链接给同事后，对方反馈无法查看或页面异常
- 下载图表数据时格式不符合预期，需要了解导出规则

### 一、Dashboard 创建与保存

#### Q1：保存 Dashboard 时，"Save" 和 "Save as Draft" 有什么区别？

**A：** 两者的区别在于 Dashboard 的发布状态：

- **Save as Draft（保存为草稿）**：Dashboard 仅保存当前配置，不会正式发布。草稿状态下，您可以随时继续编辑，但其他人通过分享链接可能无法看到最新内容（取决于是否已生成过分享链接）。
- **Save（正式保存）**：Dashboard 保存并正式发布，所有配置立即生效。如果已生成分享链接，接收方将看到最新版本。

**操作建议**：如果您还在调试图表配置，建议先使用 "Save as Draft"；确认所有图表数据正确后，再点击 "Save" 正式发布。

[截图: 顶部操作栏中 Save as Draft 和 Save 按钮的位置]

---

#### Q2：创建 Dashboard 时，"Add Chart" 和 "Use Template" 应该选哪个？

**A：** 取决于您的需求：

- **Add Chart（添加图表）**：从零开始创建一个空白图表，您需要手动选择图表类型、配置指标、设置物料范围等所有参数。适合有明确自定义需求的用户。
- **Use Template（使用模板）**：从 Widget Library（模板库）中选择一个已预配置好的图表模板。模板已包含推荐的指标和配置，您只需为图表命名并选择数据范围即可。适合希望快速搭建 Dashboard 的用户。

**操作步骤（使用模板）**：

1. 在 Dashboard 编辑页面点击 **Use Template**。
2. 在弹出的模板库中浏览或搜索模板，支持按类型筛选和预览。
3. 选择目标模板后，为图表输入名称。
4. 根据模板要求的物料层级（Material Level），选择对应的数据范围（Data Scope）。
5. 确认后，图表将自动添加到 Dashboard 中。

[截图: Use Template 模板选择弹窗]

---

#### Q3：Dashboard 最多可以添加多少个图表？

**A：** 系统对单个 Dashboard 中的图表数量没有严格的硬性上限，但添加过多图表可能会影响页面加载速度和浏览体验。建议根据实际分析需求合理规划图表数量，通常 10-20 个图表可以覆盖大多数分析场景。

---

### 二、图表类型选择与配置

#### Q4：系统支持哪些图表类型？分别适合什么场景？

**A：** Custom Dashboard 支持以下 8 种图表类型：

| 图表类型 | 适用场景 | 说明 |
|---------|---------|------|
| **Top Overview（概览卡片）** | 快速查看关键指标的汇总数值 | 以数字卡片形式展示，支持同比/环比对比，还支持目标进度和目标对比格式 |
| **Trend Chart（趋势图）** | 观察指标随时间的变化趋势 | 支持折线图和面积图，可按日/周/月切换时间粒度 |
| **Comparison Chart（对比图）** | 对比不同物料或时间段的指标差异 | 柱状图形式，支持按汇总对比、同比（YOY）、环比（POP）三种模式 |
| **Stacked Bar Chart（堆叠柱状图）** | 查看指标在不同维度上的构成 | 将单一指标按维度拆分为堆叠柱状图 |
| **Pie Chart（饼图）** | 查看指标在不同物料间的占比分布 | 只能选择一个指标，直观展示占比关系 |
| **Table（表格）** | 查看多指标的详细数据明细 | 支持半屏/全屏、二级展开、添加环比数据列 |
| **Grid Table（网格表格）** | 查看两个维度交叉的数据 | 行和列分别为不同物料维度，单元格展示交叉指标值 |
| **White Board（白板）** | 添加说明文字或插入图片 | 自由画布，可插入图片用于补充说明 |

**选择建议**：
- 想看趋势变化 → Trend Chart
- 想对比不同广告活动的表现 → Comparison Chart 或 Table
- 想看整体概况 → Top Overview
- 想看占比分布 → Pie Chart
- 想做交叉分析 → Grid Table

---

#### Q5：Trend Chart 的三种模式（多指标、多物料、自定义）有什么区别？

**A：** 三种模式决定了图表中每条线代表什么：

| 模式 | 含义 | 典型场景 |
|------|------|---------|
| **Multi Metrics（多指标模式）** | 所有指标共享同一组物料范围，每条线代表一个不同的指标 | 想在同一张图上同时看 Impressions、Clicks、CTR 的趋势 |
| **Multi Values（多物料模式）** | 只选一个指标，每条线代表一个不同的物料（如不同的 Campaign） | 想对比多个 Campaign 的 Spend 趋势 |
| **Customize（自定义模式）** | 每条线可以独立配置指标和物料范围 | 想在同一张图上看 Campaign A 的 Impressions 和 Campaign B 的 Clicks |

**注意事项**：
- 在 Multi Metrics 模式下，支持关联筛选器（Filter-linked Campaign）功能；其他两种模式不支持。
- Customize 模式的图表不支持批量创建 Dashboard。

[截图: Trend Chart 设置弹窗中的模式选择区域]

---

#### Q6：Trend Chart 的 D/W/M 快捷切换按钮是什么？

**A：** 开启 Quick Switch 功能后，图表展示区域会出现 **D（Daily/日）、W（Weekly/周）、M（Monthly/月）** 三个快捷按钮，您可以一键切换时间粒度，无需重新进入设置弹窗修改。

**开启方式**：在 Trend Chart 的 Basic Setting 面板中，勾选 **Quick Switch** 选项。

**注意事项**：当数据源为 SnS（Subscribe & Save）时，不允许开启 Quick Switch。

---

#### Q7：Comparison Chart 的 BySum、YOY、POP 三种模式分别是什么意思？

**A：**

- **BySum（按汇总对比）**：将不同物料的指标汇总值放在一起对比。例如，对比 Campaign A、B、C 各自的总 Spend。支持同时选择多个指标。
- **YOY（Year over Year，同比）**：将当前时间段的数据与去年同期进行对比。例如，今年 3 月的 Impressions 与去年 3 月的 Impressions 对比。
- **POP（Period over Period，环比）**：将当前时间段的数据与上一个时间段进行对比。例如，本月的 Clicks 与上月的 Clicks 对比。

YOY 和 POP 模式还支持自定义对比时间范围，您可以灵活设定要对比的历史时间段。

---

#### Q8：Top Overview 的三种展示格式有什么区别？

**A：** Top Overview 支持三种展示格式：

| 格式 | 说明 | 适用指标类型 | 是否需要设置目标值 |
|------|------|------------|------------------|
| **Regular（常规）** | 默认样式，展示指标的汇总数值，支持同比/环比对比 | 所有指标 | 否 |
| **Target Progress（目标进度）** | 进度条样式，展示当前值占目标值的完成百分比 | 仅数字和货币类型指标（不含百分比类指标） | 是（必填） |
| **Target Compare（目标对比）** | 目标对比样式，展示当前值与目标值的对比情况 | 仅百分比类型指标 | 是（必填） |

**Target Progress 的颜色规则**：
- 对于正向指标（如 Sales，越高越好）：完成度 ≥100% 显示绿色，80%-100% 显示橙色，≤80% 显示红色
- 对于反向指标（如 ACOS，越低越好）：完成度 ≤100% 显示绿色，100%-120% 显示橙色，≥120% 显示红色
- 对于中性指标（如 Impressions、Spend）：统一显示灰色

**注意事项**：
- Target Progress 格式下，每个 Top Overview 卡片最多选择 3 个指标。
- 目标值必须为大于 0 的数字，最多支持两位小数。
- Target Progress 支持选择未来时间范围（用于设定未来目标）。

---

#### Q9：Pie Chart 为什么只能选一个指标？

**A：** 饼图的核心用途是展示"占比分布"，即一个指标在不同物料之间的比例关系。如果同时展示多个指标，占比关系会变得混乱且难以解读。因此，系统限制饼图只能选择一个指标。

如果您需要同时查看多个指标的分布情况，建议创建多个饼图，或改用 Table（表格）类型。

---

#### Q10：Grid Table 的横向和纵向物料层级有什么搭配限制？

**A：** Grid Table 是一种二维交叉表格，行和列分别代表不同的物料维度。但并非所有维度都可以自由组合，以下是兼容关系：

| | Profile | Campaign (Parent) Tag | Campaign Type | Retailer | Share Tag |
|---|:---:|:---:|:---:|:---:|:---:|
| **Profile** | ✗ | ✓ | ✓ | ✗ | ✓ |
| **Campaign (Parent) Tag** | ✓ | ✓ | ✓ | ✗ | ✓ |
| **Campaign Type** | ✓ | ✓ | ✗ | ✗ | ✓ |
| **Retailer** | ✗ | ✗ | ✗ | ✗ | ✓ |
| **Share Tag** | ✓ | ✓ | ✓ | ✓ | ✗ |

**解读示例**：
- ✓ 表示可以组合。例如，横向选 Profile、纵向选 Campaign Tag 是允许的。
- ✗ 表示不可组合。例如，横向和纵向都选 Profile 是不允许的（同一维度不能交叉自身）。

**注意事项**：Grid Table 仅支持选择一个指标，且不支持批量创建 Dashboard。

---

### 三、物料层级与数据范围

#### Q11：什么是"物料层级"（Material Level）？

**A：** 物料层级是指您希望在图表中分析数据的粒度级别。不同的层级代表广告投放结构中的不同层次：

| 物料层级 | 含义 | 说明 |
|---------|------|------|
| **Profile** | 广告账户 | 最粗的粒度，按账户汇总数据 |
| **Campaign** | 广告活动 | 按单个广告活动查看数据 |
| **Campaign Tag / Campaign Parent Tag** | 广告活动标签/父标签 | 按标签分组查看数据，方便管理大量 Campaign |
| **ASIN / ASIN Parent Tag** | 商品/商品父标签 | 按商品维度查看数据 |
| **Advertiser** | 广告主 | DSP 平台专用，按广告主查看数据 |
| **Campaign Type** | 广告类型 | 按 SP/SB/SD 等广告类型查看数据 |
| **Placement** | 展示位置 | 按广告展示位置查看数据 |
| **Keyword** | 关键词 | 按关键词维度查看数据 |
| **Cross Retailer** | 跨零售商 | 跨平台汇总对比数据 |
| **SOV Group** | SOV 分组 | Share of Voice 分组数据（需要相应权限） |

选择不同的物料层级后，图表中的数据将按该层级进行汇总或拆分展示。

---

#### Q12：不同平台搜索商品时，为什么搜索框的提示文字不一样？

**A：** 这是因为不同广告平台对商品的命名方式不同：

| 平台 | 商品标识名称 | 搜索提示 |
|------|------------|---------|
| Amazon / DSP | ASIN | 搜索 ASIN |
| Walmart / Samsclub | Item | 搜索 Item |
| Instacart / Criteo / Target / Citrus / Kroger / Chewy / Bol / Doordash | Product | 搜索 Product |

虽然名称不同，但功能是一样的——都是搜索您投放广告的商品。您只需在搜索框中输入对应平台的商品标识即可。

### 四、Dashboard 级别筛选器

#### Q13：Dashboard Setting 中的筛选器有什么作用？

**A：** Dashboard 级别的筛选器是一种全局过滤机制，设置后会对 Dashboard 中的**所有图表**生效。目前支持以下筛选器：

- **Advertiser Filter（广告主筛选）**：仅在 DSP 平台下可用，用于筛选特定广告主的数据。
- **Campaign Tag Filter（广告活动标签筛选）**：仅在 Amazon 平台下可用，用于按标签过滤数据。
- **Market Filter（市场筛选）**：用于筛选特定市场/站点的数据。

**操作步骤**：

1. 在 Dashboard 编辑页面，点击右上角的 **Dashboard Setting** 下拉菜单。
2. 在弹出的设置面板中，根据需要配置各筛选器。
3. 保存后，Dashboard 中所有图表的数据都会按照筛选条件进行过滤。

**注意事项**：Dashboard 级别筛选器与图表自身的物料范围设置是叠加关系，两者同时生效。

[截图: Dashboard Setting 下拉菜单及筛选器配置面板]

---

#### Q14：DSP 平台的 Profile 选择为什么是两级菜单？

**A：** DSP 平台的账户结构与其他平台不同，采用 **Entity（实体）→ Advertiser（广告主）** 的两级层级结构。因此在选择 Profile 时：

1. 首先展开 Entity 层级。
2. 在每个 Entity 下查看并选择具体的 Advertiser。

选择完成后，系统会分别记录您选择的 Profile 和 Advertiser 信息，确保数据查询的准确性。

[截图: DSP 平台 Profile 选择的两级菜单]

### 五、分享与协作

#### Q15：如何分享 Dashboard 给其他人？

**A：**

**操作步骤**：

1. 在 Dashboard 列表页，将鼠标悬浮到目标 Dashboard 所在的行。
2. 悬浮后会出现操作按钮，点击 **分享** 图标。
3. 在弹出的窗口中，系统会自动生成一个 Share Link（分享链接）。
4. 点击 **复制** 按钮，将链接发送给需要查看的同事。

**注意事项**：
- 通过分享链接打开的 Dashboard 为**只读模式**，页面不会显示面包屑导航和编辑按钮，接收方无法修改配置。
- 如果 Dashboard 被删除后，访问分享链接会显示一个缺省提示页面，告知该 Dashboard 已不存在。
- 分享链接中查看的数据与正式保存的 Dashboard 一致，草稿状态的修改不会体现在分享链接中。

[截图: Dashboard 列表页的分享按钮和分享链接弹窗]

### 六、图表下载与导出

#### Q16：如何下载图表数据？

**A：**

**操作步骤**：

1. 在 Dashboard 页面中，找到您想下载的图表。
2. 点击图表右上角的 **下载** 按钮。
3. 在弹出的选项中选择导出格式：
   - **下载为 Excel**：按图表类型的固定格式导出为 Excel 文件，适合进一步数据分析。
   - **下载为图片**：导出为无底色、无标题的纯图表图片，适合用于报告或演示文稿。

### 七、批量创建 Dashboard

#### Q17：为什么有些图表不支持批量选择创建 Dashboard？

**A：** 以下类型的图表不支持通过批量选择的方式创建 Dashboard：

1. **选择了 Cross Retailer（跨零售商）的图表**：跨零售商图表涉及多平台数据整合，批量创建时无法统一处理。
2. **Trend Chart 使用了 Customize（自定义）模式的图表**：自定义模式下每条线的配置各不相同，无法批量复制。
3. **Top Overview 中选择了不同物料层级的图表**：物料层级不一致时，无法统一应用批量配置。
4. **Grid Table（网格表格）类型的图表**：Grid Table 的二维交叉配置较为复杂，不支持批量操作。

当您尝试批量创建时，如果选中的图表中包含以上类型，系统会弹窗提示具体原因，您需要取消选择这些图表后才能继续操作。

**A：** 在 Dashboard 编辑页面的右上角，可以找到 **Currency（货币）** 设置选项。点击后可选择目标货币（如 US Dollar），设置后 Dashboard 中所有涉及金额的指标都会按照所选货币单位展示。

### 九、需求反馈

#### Q19：如何向 Pacvue 提交产品需求或反馈？

**A：** Custom Dashboard 页面右下角有一个悬浮的 **Survey（需求收集）** 图标。

**操作步骤**：

1. 将鼠标悬浮到页面右下角的 Survey 图标上，图标会展开显示提示文字。
2. 点击图标，弹出需求收集表单。
3. 填写以下信息：
   - **Title（标题）**：必填，简要描述您的需求。
   - **Preferred Chart Type（期望的图表类型）**：必填，可多选。
   - **Requirement Description（需求描述）**：选填，支持富文本编辑，可详细描述您的需求。
   - **Document Upload（文档上传）**：选填，支持上传 PDF、JPEG、PNG 格式的文件，最多 5 个文件，每个文件不超过 5MB。
4. 点击提交。

提交后，Pacvue 团队会收到您的反馈并进行评估。

[截图: 页面右下角的 Survey 悬浮图标及需求收集弹窗]

## 各平台差异说明

不同广告平台在 Custom Dashboard 中的功能支持存在一定差异，以下是用户可感知的主要区别：

| 功能 | Amazon | DSP | Walmart | Instacart | 其他平台 |
|------|--------|-----|---------|-----------|---------|
| Campaign Tag 筛选 | ✓ | ✗ | 视平台而定 | 视平台而定 | 视平台而定 |
| Advertiser 筛选 | ✗ | ✓ | ✗ | ✗ | ✗ |
| Profile 选择方式 | 单级列表 | Entity → Advertiser 两级菜单 | 单级列表 | 单级列表 | 单级列表 |
| 商品搜索标识 | ASIN | ASIN | Item | Product | Product |
| Stacked Bar Chart | ✓ | ✗ | ✗ | ✗ | ✗ |
| Commerce 数据源（1P/3P） | ✓ | ✗ | ✗ | ✗ | ✗ |

**补充说明**：
- **Stacked Bar Chart（堆叠柱状图）** 目前仅 Amazon 平台支持。
- **Commerce 数据源** 分为 1P（Vendor）、3P（Seller）和 Hybrid 三种通道，主要适用于 Amazon 平台的电商数据分析。在 Table 类型图表中，选择完物料后，下方会出现数据源选择器，供您选择 1P 或 3P。
- **Cross Retailer（跨零售商）** 功能在 Pacvue HQ 模块下可用，支持 Trend Chart、Comparison Chart、Top Overview、Table、Pie Chart 五种图表类型。

## 常见问题速查

以下是用户最常遇到的问题快速索引：

---

**Q：图表配置完成后，为什么看不到数据？**

A：请依次检查以下几点：
1. **时间范围是否正确**：确认选择的开始日期和结束日期范围内有广告投放数据。
2. **Profile 是否已选择**：确认在配置中选择了正确的广告账户（Profile）。
3. **物料范围是否为空**：确认在 Scope Setting 中选择了具体的物料（如 Campaign、ASIN 等）。
4. **Dashboard 级别筛选器是否过于严格**：检查 Dashboard Setting 中的全局筛选器是否过滤掉了所有数据。
5. **货币和时区设置**：确认货币和时区设置与您的预期一致。

如果以上都确认无误仍无数据，可能是该时间段内确实没有对应的投放数据。

**Q：Comparison Chart 中的 YOY 和 POP 对比数据看起来不对，是怎么回事？**

A：请检查以下几点：
1. **对比时间范围**：YOY（同比）默认对比去年同期，POP（环比）默认对比上一个相同长度的时间段。如果您自定义了对比时间范围，请确认设置是否正确。
2. **数据可用性**：如果去年同期或上一时间段没有投放数据，对比值会显示为空或零。
3. **指标方向性**：部分指标是"反向指标"（如 ACOS，即广告花费占销售额的比例，越低越好），对比时的增减箭头颜色可能与您的直觉相反——ACOS 下降会显示为正面（绿色），上升会显示为负面（红色）。

**Q：图表上方的 Chart Tips 文案是什么意思？**

A：Chart Tips 是系统根据您的图表配置自动生成的描述文案，帮助您和查看者快速理解图表展示的内容。不同图表类型和模式下，文案格式不同。例如：

**Q：为什么有些指标在 Target Progress 格式下不可选？**

A：Target Progress（目标进度）格式仅支持**数字类型和货币类型**的指标（如 Impressions、Clicks、Sales、Spend 等），不支持百分比类型的指标（如 CTR、ACOS、ROAS 等）。这是因为进度条需要用"当前值 ÷ 目标值"来计算完成百分比，而百分比类指标本身已经是比率，再做一次百分比计算会导致含义混乱。

如果您需要为百分比类指标设置目标对比，请使用 **Target Compare** 格式。
