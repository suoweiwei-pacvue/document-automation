# Custom Dashboard 产品概览

## 功能简介

Custom Dashboard（自定义看板）是 Pacvue 平台提供的可视化数据看板工具，帮助您将分散在不同广告平台、不同层级的核心数据，集中到一个页面中进行查看和分析。

您可以根据自身的业务需求，自由选择想要关注的指标，搭配不同类型的图表组件（如数据概览卡片、趋势图、对比图、饼图、表格等），拖拽排列成最适合自己的数据看板。无论是日常监控广告表现、跨平台对比投放效果，还是制作定期汇报材料，Custom Dashboard 都能帮您高效完成。

**支持的广告平台**：Amazon、Walmart、DSP、Instacart、Citrus、BOL、Doordash 等，并支持在同一图表中跨平台对比数据。

## 适用角色

| 角色 | 典型使用方式 |
|------|-------------|
| **广告运营人员** | 搭建日常监控看板，实时追踪关键广告指标（如花费、销售额、ACOS 等），快速发现异常数据 |
| **团队主管 / 运营经理** | 创建跨平台汇总看板，纵览多个零售商平台的整体投放表现，辅助决策 |
| **品牌方 / 客户** | 通过分享链接查看看板，无需登录系统即可了解广告投放情况 |

## 使用场景

以下是 Custom Dashboard 最常见的使用场景：

1. **日常广告监控**：将最关注的广告指标（花费、销售额、ACOS、ROAS 等）集中到一个看板中，每天打开即可快速了解整体表现。
2. **跨平台效果对比**：在同一张图表中对比 Amazon 和 Walmart（或其他平台）的广告数据，直观了解各平台的投放差异。
3. **同比 / 环比分析**：通过 POP（环比，即与上一周期对比）和 YOY（同比，即与去年同期对比）功能，分析数据趋势变化，评估广告策略效果。
4. **制作汇报材料**：将图表下载为无底色图片或 Excel 文件，直接粘贴到 PPT 或报告中，省去手动制图的时间。
5. **向客户 / 团队分享数据**：生成分享链接，对方无需登录 Pacvue 即可查看看板内容，方便跨团队协作或向客户汇报。

### 一、创建看板

Custom Dashboard 支持从零开始搭建一个全新的数据看板。您可以逐个添加图表组件，也可以使用批量创建模式一次性配置多个图表，提升效率。

#### 操作步骤

1. 进入 Custom Dashboard 模块（可从 Advertising 菜单或 Executive Hub / Pacvue HQ 入口进入）。
2. 点击页面上的 **「Create Dashboard」** 按钮。

[截图: Custom Dashboard 列表页，标注 Create Dashboard 按钮位置]

3. 选择创建模式：
   - **单个创建模式**：每次添加一个图表组件，适合精细化配置。
   - **批量创建模式**：一次性配置多个图表组件，适合快速搭建看板。

4. 为每个图表组件进行配置（详见下方「配置图表组件」部分）。

5. 配置完成后进入预览页面，您可以：
   - 查看看板的整体布局效果。
   - 通过拖拽调整各图表的位置和大小。
   - 在每个图表下方确认所选的数据范围信息。

[截图: 看板预览页面，展示拖拽调整布局的操作]

6. 在页面顶部设置 **看板名称** 和 **货币单位**（Currency）。
7. 点击 **「Save」** 保存看板。

#### 配置图表组件

每个图表组件需要完成以下配置项：

| 配置项 | 说明 |
|--------|------|
| **图表类型** | 选择您需要的展示形式（见下方「图表类型说明」） |
| **图表名称** | 为图表命名，系统也支持根据配置自动生成名称 |
| **物料层级（Material Level）** | 选择数据的汇总维度，如按 Profile（账户）、Campaign（广告活动）、Campaign Tag（广告活动标签）、ASIN、ASIN Tag 等层级查看，也可选择 Cross Retailer（跨平台）模式 |
| **数据范围（Data Scope）** | 在所选层级下，选择具体要展示哪些数据对象（如选择哪些 Campaign、哪些 ASIN 等） |
| **指标（Metrics）** | 选择要展示的数据指标，如 Spend（花费）、Sales（销售额）、ACOS（广告花费占比）、ROAS（广告投入回报率）、Impressions（展示量）、Clicks（点击量）等 |
| **图表样式** | 部分图表类型支持选择展示样式，如趋势图可选择「曲线带阴影」「曲线不带阴影」或「柱状」 |

[截图: 图表组件配置弹窗，标注各配置项位置]

> **💡 小贴士：批量设置数据范围**
>
> 如果您创建了多个图表，且它们的物料层级（Material Level）相同，可以批量勾选这些图表后统一设置数据范围，无需逐个配置。但如果所选图表的物料层级不一致，系统会提示无法批量设置。
>
> 对于饼图（Pie Chart）和表格（Table），如果它们都使用了 Top X / Bottom X / Top X Mover 模式，也可以放在一起批量设置。

### 二、图表类型说明

Custom Dashboard 提供 8 种图表组件，适用于不同的数据展示需求：

| 图表类型 | 适用场景 | 说明 |
|----------|----------|------|

| **趋势图（Trend Chart）** | 观察指标随时间的变化趋势 | 以折线图或柱状图展示数据的时间趋势，支持按日（D）、周（W）、月（M）快捷切换时间粒度 |

| **堆叠柱状图（Stacked Bar Chart）** | 查看数据的构成比例 | 以堆叠柱状图展示各部分在整体中的占比（目前为 Amazon 平台专属功能） |

| **表格（Table）** | 查看详细的数据列表 | 以表格形式展示多行多列的详细数据，支持半屏和全屏两种显示模式 |
| **网格表格（Grid Table）** | 以网格形式展示数据 | 适合展示结构化的多维数据 |
| **白板（WhiteBoard）** | 添加说明文字或图片 | 自由编辑文本内容，支持插入图片，适合在看板中添加备注、说明或品牌 Logo 等 |

### 三、查看看板

创建完成后，您可以随时打开看板查看最新数据。

#### 操作步骤

1. 在 Custom Dashboard 列表页，点击您要查看的看板名称，进入看板详情页。
2. 系统会自动加载各图表组件的数据。
3. 使用页面顶部的 **日期筛选器** 调整查看的时间范围。

[截图: 看板详情页，标注日期筛选器位置]

#### 时间对比功能

Custom Dashboard 支持两种时间对比方式，帮助您分析数据的变化趋势：

- **POP（Period over Period，环比）**：将当前选择的时间段与紧邻的上一个同等时间段进行对比。例如，选择本周数据时，自动对比上周数据。
- **YOY（Year over Year，同比）**：将当前选择的时间段与去年同期进行对比。例如，选择今年 3 月数据时，自动对比去年 3 月数据。

您还可以 **自定义对比时间范围**，不受默认规则限制。例如，将今年 Prime Day 的数据与去年 Prime Day 的数据进行对比，即使两次活动的具体日期不同。

[截图: 日期筛选器展开状态，标注 POP/YOY 选项和自定义对比时间设置]

#### 趋势图时间粒度切换

在趋势图（Trend Chart）组件上，您可以通过图表上方的 **D / W / M** 快捷按钮，快速切换数据的时间粒度：

- **D（Daily）**：按天展示数据
- **W（Weekly）**：按周展示数据
- **M（Monthly）**：按月展示数据

### 八、跨平台对比（Cross Retailer）

Custom Dashboard 支持在同一个图表中展示和对比多个零售商平台的数据，帮助您从全局视角评估各平台的投放效果。

#### 操作步骤

1. 在配置图表组件时，将 **物料层级（Material Level）** 设置为 **「Cross Retailer」**。
2. 选择要对比的零售商平台（如 Amazon、Walmart 等）。
3. 选择要对比的指标。
4. 保存后，图表会按平台维度展示数据：
   - **表格（Table）**：每一行代表一个零售商平台的数据。
   - **趋势图（Trend Chart）**：每一条线代表一个零售商平台的趋势，也可以选择将多个平台的数据汇总为一条线。

## 各平台差异说明

Custom Dashboard 支持多个零售商平台的数据，大部分功能在各平台上的使用方式一致，但以下功能存在平台差异：

| 功能 | 差异说明 |
|------|----------|
| **堆叠柱状图（Stacked Bar Chart）** | 目前仅支持 Amazon 平台，其他平台暂不可用 |
| **SOV（Share of Voice，搜索份额）数据** | 仅在支持 SOV 功能的平台上可用，配置时需要选择对应的 SOV 分组和品牌/ASIN |
| **Commerce（电商市场）数据** | 包含第三方市场数据，配置时可能需要额外选择市场筛选条件（如国家、市场等） |
| **DSP 数据** | DSP 平台的数据处理方式与搜索广告有所不同，部分图表类型（如指标概览、多指标对比、仅查询汇总数据的网格表格）会跳过特定的展示优化处理，但这不影响您的正常使用 |

> **💡 提示**：在配置图表时，系统会根据您选择的平台自动过滤可用的物料层级和指标选项，您无需担心选择到不支持的配置。

## 常见问题

**Q: 我可以创建多少个 Dashboard？**
A: 系统没有对看板数量做严格限制，您可以根据需要创建多个看板，分别用于不同的监控场景（如按品牌、按市场、按广告类型等维度分别创建）。

**Q: 分享链接的接收者需要有 Pacvue 账号吗？**
A: 不需要。分享链接的接收者无需登录 Pacvue 即可查看看板内容。但请注意，分享链接中包含的是看板的只读视图，接收者无法进行编辑操作。

**Q: 看板中的数据是实时更新的吗？**
A: 看板中的数据会在您每次打开或刷新页面时重新加载。数据的时效性取决于各广告平台的数据回传速度，通常会有一定的延迟（与您在 Advertising 模块中看到的数据时效性一致）。

**Q: 为什么我在配置图表时看不到某些指标或物料层级选项？**
A: 可用的指标和物料层级选项取决于您选择的广告平台和图表类型。不同平台支持的数据维度不同，系统会自动过滤并只展示当前配置下可用的选项。如果您确认某个选项应该存在但未显示，请检查是否选择了正确的平台和图表类型。

**Q: 如何将看板中的图表用于 PPT 汇报？**
A: 点击图表上的下载按钮，选择「图片」格式导出。导出的图片默认为无底色（透明背景），可以直接粘贴到 PPT 中，与任何背景色自然融合，无需额外裁剪或去除白色背景。

**Q: 批量创建模式和单个创建模式有什么区别？**
A: 两种模式创建出的看板效果完全一致，区别仅在于操作效率。如果您需要一次性添加多个图表组件，批量创建模式可以让您在一个流程中完成所有图表的配置，避免反复进出配置弹窗。如果您只需要添加一两个图表，单个创建模式更加直观。
